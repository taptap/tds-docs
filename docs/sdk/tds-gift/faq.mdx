###  兑换参数完备，但请求兑换时返回 403。

请检查您的请求中 header 信息，是否有带上特殊的 User Agent。

###  二次校验是需要游戏方请求两次兑换接口吗？

二次校验是您按照规范开发校验接口并将地址配置后台中，由兑换系统去请求。

###  无服务器返回值的 content 参数无法解析？

返回值的 content 是一段嵌套的 json 字符串，可以先读出字符串再进行进一步 json 解析，除此之外还新增了 content_obj 字段，可以有选择的去使用。

###  如何判断兑换是否成功？

可以根据返回值中的 error 字段是否为 0 来判断。

###  无服务器兑换的 sign 字段如何使用？

可以参考文档中[签名部分](/sdk/tds-gift/guide/#签名)。

###  Unity 请求无服务器兑换接口报错 HTTP/1.1 422 Unprocessable Entity

可以通过开发者后台 -> 礼包服务中检查活动开关是否打开，以及礼包码是否失效；

### 礼包生成时出错，时间不能超过 2040 年

当前最大版本礼包时间为 2038-01-19 11:14:07
