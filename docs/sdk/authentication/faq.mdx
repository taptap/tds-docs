### 内建账户用户名和 TapTap 账户用户名不一致

目前由于 TapTap 昵称和游戏昵称是分开的两个部分，我们的设计逻辑是，第一次使用 TapTap 登录后会将登录的信息存到内建账户中，但是后续如果修改 TapTap 的昵称，内建账户中语义保存的昵称是不会更新的，如果想要使用更新后的昵称，
可以尝试游戏内新增一个[修改昵称](/sdk/authentication/guide/#设置其他用户属性)的逻辑，用户输入昵称，然后客户端调用内建账户的[设置其他用户属性](/sdk/authentication/guide/#设置其他用户属性)的方法，就可以更新昵称了。

### 内建账户修改密码后 403 (Forbidden)

如果在 `开发者中心 > 你的游戏 > 游戏服务 > 云服务 > 内建账户 > 设置` 中勾选了 密码修改后，强制客户端重新登录，那么当一个用户修改密码后，该用户的 session token 会被重置。此时需要让用户重新登录，否则会遇到 403 (Forbidden) 错误
