## 介绍

为了满足开发者在仅完成游戏核心玩法、游戏整体完成度较低情况下开启游戏先导测试的需求，同时为了保护开发者在先导测试期间的权益，TapTap 推出了**篝火测试**。

篝火测试支持隐藏游戏评分，厂商匿名等服务，开发者可以在没有评分和曝光的压力下更灵活的调整游戏设计，观测每一次变化收到的玩家反馈。

篝火测试游戏的分发途径分为 2 种：首页分发、邀请指定玩家获取测试资格。在首页推荐点击进入游戏详情页的玩家才会获取篝火测试资格，通过定向搜索游戏标题进入游戏详情页的玩家是没有测试资格的。如果需要邀请指定玩家获取测试资格，可通过链接的形式邀请。请在 [开发者中心](https://developer.taptap.com/) >> <Blue>工单</Blue> 对应的分类下按照模板规范提交测试的信息。

**特别注意**

- 参与篝火测试的游戏在开启测试后，通过以上分发途径获取测试资格的玩家，详情页展示的按钮是「试玩」。而未获取测试资格的玩家，详情页展示的按钮是「敬请期待」
- 参与篝火测试的游戏状态仅支持「敬请期待」和「测试」，玩家不可在篝火测试详情页点击预约
- 篝火测试结束后，篝火测试服的数据不支持转移到常规测试服。如需关闭篝火测试详情页，可在 [开发者中心](https://developer.taptap.com/) >> <Blue>工单</Blue> 对应的分类下提交下架申请
- 篝火测试结束后，可根据游戏后续内容迭代和实际运营需求开启新一轮篝火测试

### 参与篝火测试的条件

1. 游戏必须接入 TapTap 登录，并将 TapTap 登录作为唯一的登录方式。
2. 游戏只能在 TapTap 平台独家上线。
3. 游戏不得接入广告或以其他形式计费的内容，不支持充值功能，请勿出现任何相关的界面和功能模块。
4. 测试类型必须为删档测试。

**特别注意**

- 上述条件仅限制篝火测试期间的游戏

## 篝火测试的创建流程

1. 进入 [开发者中心](https://developer.taptap.com/) >> <Blue>创建游戏</Blue> >> 按照要求填写基础信息和基础游戏资料，游戏类型选择篝火游戏 >> 保存并提交 >> 阅读注意事项 >> 我知道了。

![](/img/campfire/创建游戏1.png)
![](/img/campfire/创建游戏2.png)
![](/img/campfire/创建游戏3.png)

2. 游戏质量审核一般需要 1-3 个工作日，如果游戏没有通过审核还请根据驳回原因调整内容，完成后可再次上传游戏内容录屏和 APK 提交审核。

![](/img/campfire/游戏质量审核.png)
![](/img/campfire/审核驳回.png)

3. 游戏通过质量审核后，进入<Blue>接入 TapTap 登录</Blue>步骤。请确认游戏开启 <Blue>应用配置</Blue> ，获取 Client ID 和 Client Secret >> 选择立即开通 <Blue>TapTap 登录</Blue> >> 根据 [《TapTap 登录功能接入文档》](/sdk/taptap-login/features/) 将 TapTap 登录和测试资格验证集成至 APK。

![](/img/campfire/接入TapTap登录1.png)
![](/img/campfire/接入TapTap登录2.png)
![](/img/campfire/接入TapTap登录3.png)
![](/img/campfire/接入TapTap登录4.png)

4. SDK 接入完成后，确认「我已接入 SDK」后跳转配置篝火测试信息。

![](/img/campfire/接入TapTap登录.png)

5. 配置完成篝火测试后点击保存，可以根据提示点击「立即前往」商店资料页补充游戏资料并提交审核。

![](/img/campfire/申请上架（篝火）.png)

**特别注意**

- 为保证玩家能够正常进入游戏，接入 TapTap 登录服务的同时，请务必接入[测试资格校验功能](/sdk/taptap-login/guide/start/#测试资格校验)
- 创建游戏时，发布地区和状态请选择「敬请期待」。篝火测试服不支持「预约」，选择「预约」状态会导致游戏被驳回审核
- 篝火测试信息首次填写保存后，变更内容请在 [开发者中心](https://developer.taptap.com/) >> <Blue>工单</Blue> 对应的分类下按照模板规范提交变更的信息
- 篝火状态选择开启时，游戏将在首页推荐中曝光。如果关闭，将停止曝光
- 用户精准度建议选择标准模式，匹配度较高时可能会减少曝光
- 开启测试前，请确认开启「篝火状态」，TapTap 登录的「用户开放范围」开放至全量用户

### 游戏品质审核标准

**游戏申请上架篝火测试前，TapTap 官方需对进行品质审核**

篝火测试欢迎不同研发阶段的游戏申请，已经实现游戏核心玩法，并确保有可供游玩的独立内容关卡，整体关卡体验内容不少于 15 分钟的游戏都可以加入我们。
但以下几种特殊情况的游戏无法通过审核：

1. 违反[《TapTap 游戏审核规范细则》](/store/store-agree/)的游戏。
2. 玩法过于单一，缺乏拓展性和可塑性的游戏。
3. 游戏调性与平台不符（如传奇类的游戏）。
4. 未获授权的二创同人类游戏。

**特别注意**

- 质量审核被驳回后，请勿在未变更内容的情况下重复提交审核申请，否则会被视为恶意扰乱审核秩序，情节严重者，游戏将不予收录

### 实名认证和防沉迷功能

参与篝火测试的游戏仍需合规接入实名认证和防沉迷功能，以下有三种实现实名认证和防沉迷功能的方式，任选其一实现即可。

1. 接入自研的实名认证和防沉迷系统。
2. 接入 [TDS 实名认证和防沉迷系统](/sdk/anti-addiction/features/)
3. 接入 [TapPlay](/sdk/tap-play/features/)

**特别注意**

- 游戏合规接入实名认证和防沉迷功能后，请在篝火测试前前往 <Blue>游戏资质</Blue> >> <Blue>未成年人防沉迷</Blue> 栏完善资质并等待上传信息审核通过
- 未接入实名认证和防沉迷功能或接入不合规的游戏，将默认接入 TapPlay
- 开启 TapPlay 前，请先前往 [TapCanary](/sdk/tap-canary/features/) 进行稳定性测试，遇到问题可在[开发者中心](https://developer.taptap.com/) >> <Blue>工单</Blue>对应的分类下联系我们

## Q&A

### 1. 篝火测试的游戏如何查看测试的数据

游戏 APK 下载量、浏览量、点击量等基础数据可在 <Blue>概览</Blue> >> <Blue>商店数据</Blue> 处查看，如需查看下载、登录游戏的玩家用户画像，或其他维度的数据信息，需要接入 [TapDB](/sdk/tapdb/features/) 功能。

### 2. 篝火测试服是否支持合并到常规测试服

不支持。  
篝火测试服是独立于常规测试服的全新页面，创建篝火测试服和常规测试服需要开发者自行创建。申请篝火测试前，不强制要求先创建常规测试服。

### 3. 篝火测试如何获得更多曝光，是否支持申请「即将上线」

不支持。  
篝火测试的模式是开放少量名额的随机限量内测，如需邀请更多的玩家加入测试，可通过链接的形式邀请，请在 [开发者中心](https://developer.taptap.com/) >> <Blue>工单</Blue> 对应的分类下按照模板规范提交测试的信息。

### 4. 篝火测试的游戏是否支持广告投放

支持。  
投放仅支持首页广告位投放，暂不支持搜索和找游戏的广告位。且仅安卓端支持投放，web 端、iOS 端、H5 均不支持。

### 5. 篝火测试的包名有什么要求

包名具有唯一性，站内不支持同时在不同页面上传相同包名的 APK。建议开发者区分「常规测试服」和「篝火测试服」的包名，如果必须使用相同的包名，请在 [开发者中心](https://developer.taptap.com/) >> <Blue>工单</Blue> 对应的分类下提交包名变更的申请。

### 6. 篝火测试如何获得内测资格

篝火测试的分发形式为两种，首页分发和邀请指定玩家获取测试资格。通过首页分发获取篝火测试资格是指玩家在首页推荐推送到游戏并能够点击进入下载。如果需要邀请玩家定向测试，需要通过链接的形式邀请玩家获取资格，请在 [开发者中心](https://developer.taptap.com/) >> <Blue>工单</Blue> 对应的分类下按照模板规范提交测试的信息。

当游戏接入测试资格校验功能后，请在<Blue>游戏服务</Blue> >> <Blue>应用配置</Blue> >> <Blue>生态服务</Blue> >> <Blue>TapTap 登录</Blue> 添加测试用户白名单或开放全量用户，然后再申请邀请篝火测试的链接。

**请注意**，邀请链接中的测试时间仅为该链接可使用的有效期限。下载游戏需要将游戏的 <Blue>发布地区和状态</Blue> 调整为测试。当篝火测试游戏的详情页状态为待上架或屏蔽时，通过邀请链接获取篝火测试资格的玩家也无法下载游戏。
