##  1. 游戏入库
1. 进入 [开发者中心](https://developer.taptap.com/) >> <Blue>创建游戏</Blue> >> 按照要求填写基础信息和基础游戏资料即可完成入库。  
2. 未通过上架审核的游戏可通过 对应游戏右下角的“┆”按钮进行重新提交或删除。  
3. 游戏提交审核后，在相关资料核实无误的情况下 TapTap 官方预计会在 5 个工作日内通过审核，审核通过后玩家即可在 TapTap 平台搜索到您的游戏详情页。

**特别注意**
- 创建游戏后，游戏为「待上架」状态（未上架）。发布游戏需要点击「申请上架」到 TapTap 平台进行审核
- 游戏在待上架阶段可以使用「商店」、「用户系统」、「游戏服务」和「数据分析」的部分功能，但是无法使用「运营」功能  
- 游戏待上架阶段可以先上传 APK，但「游戏服务」的部分功能需要上传 APK 并通过审核后才能使用
- 待上架游戏的上限为 10 个，超过将无法创建新游戏

## 2. 游戏预约
1. 游戏上架后，由厂商根据自己的游戏宣发节点选择是否需要开启“预约”。  
2. TapTap 建议游戏入库同时开启预约，平台玩家会主动选择预约感兴趣的游戏，这样浏览过详情页的玩家可立即转化为预约量。

## 3. 游戏更新

1. 游戏申请上架审核通过后，原“申请上架”按钮将变为“更新游戏”，游戏后续的素材和 APK 更新维护可通过此按钮提交到 TapTap 平台进行审核。  
2. 游戏更新提交审核后，在相关资料核实无误的情况下 TapTap 官方人员预计会在 1 个工作日内通过审核。

**特别注意**
- 游戏 APK 包名变更、版本号倒退等情况，可能导致无法上传 APK 。包名变更会使之前安装过的玩家出现存档丢失，重新安装等情况。如确定需要修改，我们建议发布相关公告后通过 [开发者中心](https://developer.taptap.com/)  >> <Blue>工单</Blue> 中对应的分类下提交申请
- 创建游戏后，可在 <Blue>全部游戏</Blue> 页面可以查看创建过的所有游戏，点击游戏标题即可进入对应游戏的面板
- 厂商也可点击页面顶部「游戏名称 - <Blue>全部游戏</Blue> 」返回查看全部已创建游戏

## 4. 游戏测试或上线
1. TapTap 为您提供一站式自助功能，游戏如需开启测试或上线可通过 <Blue>商店</Blue> >> <Blue>更新游戏</Blue> >> <Blue>商店设置</Blue> 中自行修改对应状态提交审核，审核通过后游戏状态将变更为您选择的状态。  
2. 当游戏进入人数到达饱和需要关闭本次测试，或上线后因不可抗力需要关闭下载。均可通过 <Blue>商店</Blue> >> <Blue>更新游戏</Blue> >> <Blue>商店设置</Blue> 中自行修改为“预约”或“敬请期待”状态提交审核。请悉知由于测试服不开启预约功能，选择提交变更的状态会默认保持为“敬请期待”。  
3. 游戏状态变更提交审核后，在相关资料核实无误的情况下 TapTap 官方人员预计在 1 个工作日内通过审核。  
4. 游戏如需在特定的时间节点测试或上线，请至少提前 48 小时在  [开发者中心](https://developer.taptap.com/)  >> <Blue>工单</Blue> 中对应的分类下按模板规范提交信息，我们将根据您同步的信息安排变更游戏状态。后续您自行提交“更新游戏”时，请检查“商店设置”中的状态是否为当前所变更的状态，以避免当前游戏状态被提交的更新覆盖。  
5. iOS 端测试除了进行上述操作时，还需将“TestFlight 外部链接”同步给我们为您配置，否则将无法正常开启测试，您同样可在 [开发者中心](https://developer.taptap.com/)  >> <Blue>工单</Blue>  中对应的分类下按模板规范提交该信息。  
6. 如果游戏 iOS 端未使用“TestFlight 外部链接”，TapTap 为您提供了邮箱邀请方式。具体流程为我们会向您发放一批邮箱，在您收到邮箱后请录入到 TestFlight 后台，并对这批邮箱进行资格邀请，我们在确认收到这批邀请后，您即可通过上述操作开启 iOS 端的测试。该需求也请在 [开发者中心](https://developer.taptap.com/)  >> <Blue>工单</Blue>  中对应的分类下按模板规范提交该信息。

**特别注意**
- 游戏状态变更需在符合“TapTap 游戏审核规范细则”和“国家相关规范”的前提下才能通过审核（如：无版号游戏提交“开放下载”将无法正常变更）
- 请确保游戏在变更状态前的 24 小时内已正确提交相关资质和 APK 并已通过审核  
- iOS 端的用户需通过点击网页版 TapTap 游戏详情页的“试玩”按钮申请测试，系统将会引导他安装TestFlight，若已经安装会唤起TestFlight 软件直接获取下载权限

## 5. 测试服配置
1. 开发者通过 [开发者中心](https://developer.taptap.com/) 入库的页面为正式服，如需申请测试服可在 [开发者中心](https://developer.taptap.com/)  >><Blue>工单</Blue> 中对应的分类下提交申请。  
2. 测试服是基于正式服专门创建的进行游戏测试用的页面，测试服与正式服的评分会分别计算，可在一定程度上避免游戏首发时因前期测试导致的低分影响后期运营宣发。  
3. 测试服不单独提供预约功能，游戏在测试服进行测试时，正式服仍提供预约功能，在测试期间也可为游戏积累预约用户。游戏开测时会将正式服的预约玩家引导至测试服进行测试。

## 6. 预约里程碑
1. 预约里程碑用来帮助游戏吸引玩家进行预约。可在 <Blue>游戏运营</Blue> >> <Blue>预约里程碑</Blue> 中设置预约里程碑并提交审核，审核通过后将在游戏详情页展示预约里程碑。  
2. 预约里程碑最多三档，需填写内容及要求：
- 达成里程预约数；
- 奖励内容：限 25 字符；
- 奖励图片：透明背景 200 x 200px，不超过 100KB，格式必须为 PNG。  
3. 预约里程碑发布后不可自行修改，请确认后再提交审核。如因提交错误或不可抗力需修改预约里程碑，请在论坛发布预约里程碑修改公告，公告内容需要包括：新达成奖励数、新奖励内容、预约里程碑需修改原因等。公告发布后，可在 [开发者中心](https://developer.taptap.com/)  >><Blue>工单</Blue> 中对应的分类下提交修改申请。

## 7. 激活码和礼包码
1. 激活码功能是在游戏需进行限量测试前配置，您可在对应游戏下的 <Blue>游戏运营</Blue> >> <Blue>激活码&礼包码</Blue> >> <Blue>新增游戏码</Blue> 中设置，完成后可通过 [开发者中心](https://developer.taptap.com/)  >><Blue>工单</Blue> 对应的分类下按模板规范提交需限量测试信息，我们将根据您同步的信息安排限量测试。  
2. 礼包码功能是在游戏测试或上线前配置，您可在对应游戏下的 <Blue>游戏运营</Blue> >> <Blue>激活码&礼包码</Blue> >>   <Blue>新增游戏码</Blue>  中设置，完成后玩家可通过详情页领取对应的礼包码在游戏内兑换使用。

**特别注意**
- 激活码&礼包码上传成功后会在您的开发者中心显示未“待审核”，我们会在您同步的时间进行发放后才变更为“发放中”。码的发放规则均为先到先得，该功能仅支持安卓端可使用。

## Q&A

### 1. TapTap 是所有的游戏都收录吗？
详情可参考 [TapTap游戏审核规范细则](/store/store-agree#暂不收录游戏/)

可以。  
请于 <Blue>工单</Blue> 联系我们，并说明原因。

### 6. Apk 置为当前时间设置后，游戏为什么没有在设置的时间开放下载？
APK 置为当前的设置不会改变游戏状态，仅为变更当前提供下载的 APK。

### 7. 游戏名称可以修改吗？
可以。  
如需修改游戏名称，可在该游戏下的 <Blue>商店</Blue> >> <Blue>游戏资料</Blue> 中修改后提交审核。

修改游戏名称时，请统一把游戏图标、推广图、截图等资料上的名字一起修改，同时游戏名必须与相关资质保持一致的修改才能通过审核。我们建议您在修改名称后在开发者的话或论坛中公告玩家游戏原名称以及修改游戏名称的原因，以免给玩家带来不良游戏体验，影响游戏评分。

### 8. TapTap 收录没有版权但资质齐全的游戏吗？
TapTap 非常重视版权问题，没有版权的游戏一律不做收录。

### 9.  预约里程碑内容有限制吗？
预约里程碑需对奖励内容进行清晰说明，不可用例如 “ 预约大礼包 ” 等字样表示。预约里程碑不可涉及任何实物或金钱性质内容（包括且不限于现金、礼品）。

### 10.  如何向玩家发放预约里程碑奖励？
预约里程碑奖励一般在游戏首发后向预约玩家发放，建议在游戏内通过站内信的形式直接向全服玩家发放。

如需通过礼包码的形式发放，请于 <Blue>游戏运营</Blue> >> <Blue>激活码&礼包码</Blue> >> <Blue>新增游戏码</Blue> 中设置，并在 <Blue>工单</Blue> 联系我们配置发放。

### 11. 事故停服保护测试服可以开启吗？
不可以。  
该功能仅用于正式服，测试服已作为一种保护机制无法再次开启该功能。

### 12. 如何使用发布状态变更？
请从 <Blue>商店</Blue> >> <Blue>更新游戏</Blue> >> <Blue>商店设置</Blue> 中修改状态提交审核。
