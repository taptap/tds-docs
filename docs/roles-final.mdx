客服模块拥有与开发者中心部分独立的权限系统。权限由角色与范围两部分组成。

- 角色：决定用户能使用哪些功能，进行哪些操作；
- 范围：决定用户能访问的工单。

### 角色

按照不同的职责，系统内置了以下几种角色：

| 用户角色   | 职责                                                                     | 访问路径   | 计费   |
| ---------- | :----------------------------------------------------------------------- | ---------- | ------ |
| 厂商管理员 | 维护服务与安全相关配置                                                   | 开发者中心 | 不计费 |
| 管理员     | 日常管理<br/><small>拥有除了服务与安全相关配置之外的所有配置权限</small> | 客服工作台 | 不计费 |
| 客服       | 直接与用户沟通，处理客户诉求                                             | 客服工作台 | 计费   |
| 协作者     | 协助客服处理用户诉求                                                     | 客服工作台 | 不计费 |
| 开发者     | 查看、维护游戏接入与运营过程中时关心的配置                               | 客服工作台 | 不计费 |

厂商管理员是在开发者中心拥有客服权限的 Tap 用户。其他角色的用户是独立的客服系统用户，通过独立于开发者中心的客服工作台登录使用客服模块，我们统称他们为「成员」。成员可以由厂商管理员在开发者中心添加，也可以由管理员在客服工作台添加。一个成员可以拥有一种或多种用户角色，厂商管理员或管理员可以修改成员的角色。

### 范围

如果用户拥有「访问工单」的权限，其能访问的工单有如下三种不同的范围：

- 所有工单
- 仅限其所在组的的工单
- 仅限分配给该用户的工单
