客服模块拥有与开发者中心部分独立的角色与权限系统。用户按照不同的职责可以分成以下几种角色：

| 用户角色   | 职责                                                                 | 访问路径   | 计费   |
| ---------- | :------------------------------------------------------------------- | ---------- | ------ |
| 厂商管理员 | 维护服务与安全相关配置                                               | 开发者中心 | 不计费 |
| 管理员     | 日常管理<br/><small>拥有除了服务与安全相关配置之外的所有权限</small> | 客服工作台 | 计费   |
| 客服       | 处理客户诉求                                                         | 客服工作台 | 计费   |
| 协作者     | 协助客服处理用户诉求                                                 | 客服工作台 | 不计费 |
| 开发者     | 查看或维护项目组接入服务时关心的配置                     | 客服工作台 | 计费   |

厂商管理员是在开发者中心拥有客服权限的 Tap 用户。其他角色的用户是独立的客服系统用户，通过独立于开发者中心的客服工作台登录使用客服模块，我们统称他们为「成员」。成员可以由厂商管理员在开发者中心添加，也可以由管理员在客服工作台添加。一个成员只能属于一种用户角色，厂商管理员或管理员可以切换成员的角色。

其中客服又细分为以下角色：

| 客服角色 | 权限                   |
| -------- | :--------------------- |
| 主管     | 处理所有工单           |
| 组员     | 处理其所在组的的工单   |
| 专员     | 处理分配给该用户的工单 |

协作者细分为以下角色：

| 协作者角色 | 权限                             |
| ---------- | :------------------------------- |
| 主管       | 查看所有工单、提交内部留言       |
| 组员       | 查看其所在组的工单、提交内部留言 |
